  How the Mobile Preview Would Work (Step by Step)

  1. Sir says: "VoxKage, build me a landing page for my portfolio"
  2. Mobile app sends the request to the FastAPI server, which forwards it to
  OpenCode Zen API (same Claude Code session that powers me)
  3. Claude Code generates the HTML/CSS/JS code and returns it
  4. Mobile app receives the code, displays it in the code editor panel, and
  simultaneously injects it into the WebView preview pane
  5. Sir taps "Preview" or it auto-updates вҖ” the iframe renders the site in
  real-time, totally sandboxed
  6. Sir says: "Change the font to serif" вҖ” VoxKage edits the code, preview
  updates instantly
  7. Sir taps "Save to Library" вҖ” the project (code + metadata) is uploaded to
  the shared backend
  8. Later, on laptop: "VoxKage, open the portfolio project from yesterday"
  вҶ’ I pull it from the shared database, load it into my context, and start
  editing

  LiveCodes SDK (The Secret Weapon)

  This is the most interesting find. LiveCodes is:
  - MIT licensed вҖ” free, open-source
  - Client-side only вҖ” no server needed for the preview
  - 90+ languages вҖ” HTML, CSS, JS, React, Vue, Svelte, TypeScript, Python, Go,
  Ruby, PHP...
  - Mobile-friendly вҖ” works on touch
  - Has a JS/TS SDK, React SDK, Vue SDK вҖ” npm i livecodes
  - Export/Import вҖ” projects can be saved as JSON, shared via URL
  - Sandboxed вҖ” security is built-in

  You can literally do:
  import { createPlayground } from 'livecodes';

  createPlayground('#container', {
    config: {
      markup: {
        language: 'html',
        content: '<h1>Hello World</h1>',
      },
      css: 'h1 { color: red; }',
      js: 'console.log("hello")',
    },
  });
  And it renders a full playground with editor + live preview in that div. Embed
  this in a WebView and you've got your mobile playground.

  The Database Sync Pattern

  The simplest architecture that works:

  1. Server stores projects as JSON blobs: { id, name, html, css, js, version, 
  updated_at, user_id }
  2. Mobile app pushes/pulls via REST endpoints:
    - POST /projects вҶ’ save a new project
    - GET /projects вҶ’ list all projects
    - GET /projects/:id вҶ’ load full project
    - PUT /projects/:id вҶ’ update project
    - POST /projects/:id/sync вҶ’ conflict resolution (last-write-wins or manual
  merge)
  3. Desktop VoxKage (me) accesses the same API:
    - voxkage pull-project portfolio вҶ’ downloads the code into my context
    - voxkage push-project portfolio вҶ’ uploads my changes back
  4. Version history вҖ” each save creates a new version entry so you never lose
  work. The server keeps the last 10 versions.
