## 10. VISUAL IMMERSION & COMPONENT ENHANCEMENTS (NEW DISCUSSION)

This section maps out the proposed architecture for rendering Matplotlib charts, fetching authentic images for free, and maintaining a strict 60/40 component-to-text ratio to deliver a visual-first chat interface.

### A. Matplotlib Rendering Failure: Root Cause & Solution
* **The Problem:** The `run_matplotlib_script` tool executes perfectly and returns a base64 markdown image: `![title](data:image/png;base64,...)`. However, `MarkdownRenderer.tsx` lacks parsing for `![alt](url)` markdown images. As a result, the base64 string is rendered as plain text or ignored entirely.
* **The Solution:** Update `renderTextWithInlineFormatting()` and `renderInlineSpans()` in the frontend parser to match the markdown image syntax:
  ```typescript
  const imageRegex = /!\[(.*?)\]\((.*?)\)/g;
  ```
  When matched, map it to a custom `MarkdownImage` component styled to display base64 data URIs and external links natively with a dark-background container card.

### B. Free & Authentic Image Fetching Strategies (Zero Cost API)
To fetch authentic images for entities (e.g. fish species, historical figures, cities) instead of generic placeholders:
1. **DuckDuckGo Image Scraper (Dynamic Backend Fetching)**:
   - Create a backend utility tool (e.g., `fetch_image_urls(query: str, count: int = 5) -> List[str]`).
   - Query DuckDuckGo's vqd/JSON endpoint (e.g., `https://duckduckgo.com/d.js` or scraper endpoints) to dynamically parse clean, unauthenticated image URLs directly from the web during tools execution.
2. **Wikipedia & Wikimedia Commons API (Unauthenticated & Free)**:
   - Wikipedia's media API is free, fast, and does not require an API key.
   - For a query like *"Gobiocichla wonderi"*, query `https://en.wikipedia.org/w/api.php?action=query&prop=pageimages|pageterms&titles=...&format=json&piprop=original` to retrieve the original high-resolution photo from Wikimedia.
3. **Integration in Agent Loop**:
   - The backend Agent can automatically chain a `fetch_images` step when compiling lists of visual subjects, passing the URLs directly into a `<Carousel images="url1,url2..." />` custom component.

### C. The 60% Components / 40% Text Ratio Enforcement
To stop VoxKage from relying on generic text walls, we will update the system prompt in `backend/system_prompt.py`:
* **The 60/40 Composition Rule**: Enforce that at least 60% of the screen space (components ratio) must consist of visual UI widgets:
  - `<Carousel images="..." />` for galleries or lists of subjects.
  - `<Chart type="bar|line" data="..." labels="..." />` for statistics/comparisons.
  - `<TaskList items="..." />` for progressive operations.
  - Collapsible `<Table>` layouts for data properties.
* **Conciseness Directive**: Plain text paragraphs are restricted to brief summaries and transitions, representing at most 40% of the response.

### D. Immersive UI Enhancements: Lightbox & Zoom Transitions
1. **Interactive Image Modal (Lightbox)**:
   - Wrap all images (standalone markdown images and carousel slides) in a `TouchableOpacity`.
   - Clicking an image opens a full-screen React Native `Modal` displaying a dark overlay.
   - Use React Native's `Animated` library to scale the image up smoothly.
2. **Meta Details & Source Linking**:
   - Underneath the zoomed image, display the title, description, and an editorial "Source Link" button.
   - Example:
     ```
     ┌──────────────────────────────────────────────────┐
     │                      [Image]                     │
     │                                                  │
     │  "Gobiocichla wonderi (Wonder's cichlid)"        │
     │  Discovered in the Niger River basin.            │
     │                                                  │
     │  [ Visit Wikipedia Source ]                      │
     └──────────────────────────────────────────────────┘
     ```
3. **Smooth Swipe Carousel**:
   - Improve the existing `CarouselComponent` with smoother scroll gesture response, snapping physics, and clear dot page indicators.
